HTML5 Semantic Elements

A semantic element clearly describes its meaning to both the browser and the developer.

Examples of non-semantic elements include <div> and <span> which do not convey any meaning about their content.

Examples of semantic elements include <form>, <table> and <article> which clearly define their content.

Browser Support

HTML5 semantic elements are supported in all modern browsers.
Additionally, you can "teach" older browsers how to handle "unknown" elements.
Read about it in the HTML5 browser support documentation.

New Semantic Elements in HTML5

HTML5 introduces new semantic elements for defining different parts of a web page:

  • <section>
  • <article>
  • <header>
  • <footer>

HTML5 <section> Element

The <section>element defines a section in a document.

According to the HTML5 specification, "a section is a thematic grouping of content, typically with a heading."

A home page can be divided into sections for introduction, content, and contact information.

HTML5 <article> Element

The <article> element defines independent, self-contained content.

An article should make sense on its own and should be able to be read independently from other parts of the website.

Examples of where the <article> element can be used include

  • forum posts
  • blog articles
  • newspaper articles

HTML5 <header> Element

The <header> element sets a header for the document or section.

The <header> element should be used as a container for introductory content.

HTML5 <footer> Element

The <footer> element specifies a footer for the document or section.

The footer typically contains information such as the document author, copyright information, links to terms of use, contact information, and so on.

There can be multiple <footer> elements in one document.
